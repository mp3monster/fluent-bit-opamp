OpAMP Server UI

Help

This console monitors connected agents, highlights late communications, and lets you queue commands or staged configurations.

  • Amber rows are past the delayed comms threshold.
  • Red rows are past the significant comms threshold.
  • Rows with pending config or commands show a colored border (pending indicator).
  • The pending indicator means the server has queued a change for that agent but has not yet sent it.
  • Until a client reports service.instance.id, the UI uses the OpAMP unique ID (instance_uid) as the service instance name/display ID.
  • Disconnected status means the agent explicitly sent an OpAMP disconnect message and will be purged after the configured retention period.
  • Disconnected agents are excluded from late/overdue counts and are tinted electric blue in the table.

Main Page Right-Click Options

On the main client table, right-click a client row to open the context menu.

  • Remove Client: Removes that client record from provider memory.
  • Issue New Unique ID: Requests a new OpAMP instance UID for that client.

Global Settings Overview

Use the Global Settings button in the main header to open provider-wide settings. The dialog has two tabs: ServerSettings and ClientSettings.

  • ServerSettings tab: Communication thresholds and provider event-history size.
  • ClientSettings tab: Default client heartbeat frequency managed by the provider.

Global Settings Save Actions

  • OK: Applies changes from the active tab to live provider settings.
  • OK + Save: Applies server settings and persists them to opamp.json.
  • Cancel: Closes the dialog without applying pending edits.

Note: Whether a client permanently stores heartbeat-related changes is client-specific behavior. The server can request/track heartbeat settings, but client-side persistence across restarts depends on the client implementation.

Provider Config Editability

The list below shows which provider configuration values are currently editable directly in the UI.

Security note: Not all server settings are exposed for UI control. Some values remain configuration-file or deployment managed to reduce operational and security risk.

  • delayed_comms_seconds Editable in UI
  • significant_comms_seconds Editable in UI
  • client_event_history_size Editable in UI
  • default_heartbeat_frequency Editable in UI
  • webui_port Not editable in UI
  • minutes_keep_disconnected Not editable in UI
  • retryAfterSeconds Not editable in UI
  • log_level Not editable in UI

Use the main page to open an agent, paste config/commands, and save. Changes are queued until the next agent check-in.
